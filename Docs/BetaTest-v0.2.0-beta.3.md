# Game Translator Lens v0.2.0-beta.3 测试说明

## 本版修复重点

- 翻译框调整体验优化：点击“显示翻译框”只预览位置，点击“调整翻译框”才进入可移动/缩放模式。
- 翻译框拖动不再使用 Windows 原生窗口拖动，减少贴边半屏/最大化吸附干扰。
- 调整完成后点击“完成调整”，程序会自动恢复鼠标穿透，避免忘记关闭调整模式。
- 设置自动保存：透明度、字号、鼠标穿透、回话输入条、翻译服务、API URL、API Key、模型都会自动保存并应用；API Key 使用当前 Windows 用户 DPAPI 保护保存。
- 增加 beta 测试入口：打开数据目录、打开日志、打开去重日志、导出诊断、清除本机数据。
- 增加运行日志、崩溃日志和可选去重调试日志：`%AppData%\GameTranslatorLens\runtime.log`、`%AppData%\GameTranslatorLens\crash.log`、`%AppData%\GameTranslatorLens\dedupe.log`。
- 内置一组默认聊天区域和翻译框位置；测试者可以直接开始，但不同分辨率或 UI 缩放下仍建议重新框选。
- 如果打开聊天历史但没有新消息，翻译框会临时显示最近翻译约 5 秒，方便回看历史。
- 去重增强：使用玩家名模糊匹配、空格无关整句相似度、token 覆盖、字符 bigram、最长公共子序列、编辑距离、数字/时间强锚点和前后邻居锚点评分。
- 去重逻辑不再按测试样例硬编码普通韩语错字；普通聊天靠通用相似度，游戏稳定术语仍由术语表/短句库处理。

## 推荐配置

- 显示模式：无边框窗口或窗口化无边框，不建议独占全屏。
- OCR 引擎：OneOCR。
- OCR：OneOCR 自动识别；当前本地接口不支持强制指定英语、日语或韩语。
- 翻译：DeepSeek 或 OpenAI Compatible API。
- DeepSeek API URL：`https://api.deepseek.com`
- DeepSeek 模型：优先 `deepseek-v4-flash`。

## DeepSeek API

1. 打开 `https://platform.deepseek.com/api_keys`。
2. 登录并创建 API Key。
3. 在本程序选择 DeepSeek，填写 API URL 和 API Key。
4. 点击“获取模型”，选择 `deepseek-v4-flash`。

## 框选区域

- 点击“选择聊天区域”。
- 框选 游戏左侧聊天消息文本框部分。
- 需要完整包含 `[玩家名]：正文` 这一整行。
- 不要框太大，尽量避开头像、HUD、菜单和背景噪声。
- 当前版本带默认区域；如果翻译不动、漏字或位置不对，第一步先重新框选。

## 翻译框

- 点击“显示翻译框”后可以预览翻译框。
- 点击“调整翻译框”进入调整模式，拖动顶部横条移动，拖动右下角缩放。
- 调整完成后点击“完成调整”，程序会恢复鼠标穿透。
- 默认开启鼠标穿透，不挡游戏操作；关闭鼠标穿透后也可以滚动翻译框历史。
- 底部回话输入条默认显示，可在主窗口关闭；输入中文回车后会翻译到选择的目标语言并复制到剪贴板。
- 回话热键默认关闭，可选开启并选择预设组合。
- 拖动或调整大小后会自动保存。
- 暂停会隐藏翻译框，但不清空历史。
- “清空”按钮会清空主日志和翻译框历史。

## Beta 测试入口

- “打开数据目录”：打开 `%AppData%\GameTranslatorLens`，里面有设置、日志和诊断文件。
- “打开日志”：打开 `runtime.log`，记录程序内实时记录。
- “记录去重调试日志”：默认关闭；测试重复/漏翻问题时开启，会记录 OCR 聊天行、锚点判断、候选新消息和重复丢弃原因。
- “打开去重日志”：打开 `dedupe.log`。
- “导出诊断”：生成 `diagnostics-日期时间.txt`，包含脱敏配置、最近日志、崩溃日志和去重日志尾部；API Key 不会明文导出。
- “清除本机数据”：清空设置、API Key、日志、诊断文件和 overlay 历史，并恢复默认配置。

## 如果出问题

- 如果程序闪退、卡住或翻译没反应，优先点击“导出诊断”，把生成的 `diagnostics-*.txt` 发给开发者。
- 如果程序直接打不开，可以手动发送 `%AppData%\GameTranslatorLens\crash.log` 和 `%AppData%\GameTranslatorLens\runtime.log`。
- 如果重复/漏翻，打开“记录去重调试日志”，复现几分钟后再“导出诊断”。
- 如果翻译没反应，先确认 API Key、模型、聊天区域和 无边框模式。
- 如果 OCR 识别不稳定，优先重新框选聊天区域并导出诊断；当前 OneOCR 自动识别，不支持强制指定源语言。
