# QuickStartWindow 界面设计方案

三个方案都是可直接替换的 XAML 文件，核心功能完全不变（所有事件处理、图片引用、交互行为一致）。

## 如何测试

```bash
# 1. 备份当前文件
cp QuickStartWindow.xaml QuickStartWindow.xaml.bak

# 2. 复制想测试的方案
cp Design-Proposals/方案A-侧边导航式.xaml QuickStartWindow.xaml

# 3. 编译运行
dotnet build Game-Translator-Lens.csproj -c Release
# 然后启动程序，点击"快速上手"
```

## 三方案对比

| 维度 | 方案 A · 侧边导航 | 方案 B · 卡片网格 | 方案 C · 沉浸轮播 |
|------|-------------------|-------------------|-------------------|
| **布局** | 左栏 268px 导航 + 右内容区 | 2×3 卡片网格全览 | 单步沉浸式，一次一步 |
| **色系** | 深蓝黑 + 蓝色 (#0A84FF) | 暖炭黑 + 琥珀金 (#E8950A) | 纯黑 + 霓虹青 (#00E5FF) |
| **风格** | 专业 App 风格 | Dashboard 仪表盘 | 游戏感 / 电竞风 |
| **窗口** | 1200×820 | 1260×920 | 1120×780 |
| **导航** | 点击左侧步骤名切换 | 滚动浏览全部卡片 | 点击底部圆点切换 |
| **图片** | 每步 1 张大图 | 每卡 1-2 张小缩略图 | 大幅 hero 图 |
| **适合** | 初次用户按顺序操作 | 回访用户快速定位 | 视觉冲击、展示用 |

## 方案 A — 侧边导航式

```
┌──────────┬──────────────────────────────────┐
│  LOGO    │  步骤标题                         │
│  快速上手 │  说明文字                         │
│          │  ┌──────────────────────────┐    │
│ ● 1 设置 │  │      截图大图              │    │
│   游戏窗口│  │                          │    │
│ ○ 2 API  │  └──────────────────────────┘    │
│ ○ 3 配置 │                                  │
│ ○ 4 翻译框│  ☐ 下次不再提示    [开始配置]     │
│ ○ 5 效果 │                                  │
│ ○ 6 回话 │                                  │
└──────────┴──────────────────────────────────┘
```

- 左侧深色导航栏，选中项蓝色高亮 + 左边框
- 步骤编号圆圈选中时变蓝底白字
- 内容区滚动查看，标题栏可拖拽

## 代码说明

- 所有方案共用 `QuickStartWindow.xaml.cs`（代码后置不变）
- 引用元素名保持一致：`DoNotShowAgainCheck`、`GuideScrollViewer`
- 事件处理器一致：`Close_Click`、`TitleBar_MouseLeftButtonDown`、`GuideScrollViewer_PreviewMouseWheel`、`GuideImage_MouseLeftButtonUp`
- 图片路径一致：`Resources/QuickStart/*.png`
- 方案 A/C 使用 RadioButton 切换内容（纯 XAML 实现，无需新事件处理）
